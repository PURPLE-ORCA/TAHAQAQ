# i18n Guide — TAHAQAQ Mobile

## Overview

| Aspect | Detail |
|--------|--------|
| **Library** | `i18n-js` (~6 KB gzipped) + `expo-localization` |
| **Languages** | English (`en`), French (`fr`), Arabic (`ar`) |
| **RTL support** | Arabic triggers `I18nManager.forceRTL(true)` + app restart |
| **Translation keys** | 214 keys across 10 feature namespaces |
| **Persistence** | `expo-secure-store` key `app-locale` |
| **Reactivity** | Custom event emitter hook (not React Context) |

## Architecture

### Why event emitter over Context?

| Concern | Context approach | Event emitter approach |
|---------|-----------------|----------------------|
| Provider nesting | Must wrap every route group | None — singleton, module-level |
| Re-render scope | All consumers re-render | Only components calling `useI18n()` |
| Extra bundle | react-i18next ~71 KB | i18n-js ~6 KB |
| Flat-file usage | Impossible without Context | `import { t } from '@/locales'` works everywhere |

### Data flow

```
User taps language option
        │
        ▼
setLocaleAppWide(locale)          ← src/hooks/useI18n.tsx
   │
   ├─► setI18nLocale(locale)      ← updates i18n.locale singleton
   ├─► SecureStore.setItemAsync() ← persists to expo-secure-store
   ├─► I18nManager.allowRTL()     ← toggles RTL if locale is 'ar'
   └─► notifyListeners()          ← calls every function in the Set<Listener>
              │
              ▼
   Every useI18n() component
   forceUpdate((n) => n + 1)
              │
              ▼
   t('key') reads live singleton
   → new translated strings render
```

## Key Files in Detail

### `src/locales/index.ts` — Singleton

```ts
import { I18n } from "i18n-js";
import { getLocales } from "expo-localization";
import * as SecureStore from "expo-secure-store";

export const i18n = new I18n({ en, fr, ar });

// Init: stored locale → device locale → fallback to 'en'
const stored = SecureStore.getItem(LOCALE_KEY);           // synchronous at init
const deviceLang = getLocales()[0]?.languageCode ?? "en";
const initial = stored && SUPPORTED.includes(stored)
  ? stored
  : SUPPORTED.includes(deviceLang) ? deviceLang : "en";

i18n.locale = initial;
i18n.enableFallback = true;
i18n.defaultLocale = "en";

// Flat exports — usable outside React
export const t = (key: string, options?: Record<string, unknown>) =>
  i18n.t(key, options);
export const getCurrentLocale = () => i18n.locale;
export const setLocale = (locale: string) => { i18n.locale = locale; };
```

### `src/hooks/useI18n.tsx` — React hook

```tsx
const listeners = new Set<Listener>();

export function setLocaleAppWide(newLocale: string) {
  setI18nLocale(newLocale);                                    // update singleton
  SecureStore.setItemAsync(LOCALE_KEY, newLocale).catch(() => {}); // persist
  // RTL handling
  const isRTL = newLocale === "ar";
  if (isRTL) { I18nManager.allowRTL(true); I18nManager.forceRTL(true); }
  else       { I18nManager.allowRTL(false); I18nManager.forceRTL(false); }
  notifyListeners();                                          // force re-renders
}

export function useI18n() {
  const [, forceUpdate] = useState(0);
  useEffect(() => {
    const listener = () => forceUpdate((n) => n + 1);
    listeners.add(listener);
    return () => { listeners.delete(listener); };
  }, []);
  const setLocale = useCallback((newLocale: string) => {
    setLocaleAppWide(newLocale);
  }, []);
  const locale = getCurrentLocale();
  return { t: i18n.t.bind(i18n), locale, setLocale, isRTL: locale === "ar" };
}
```

### `src/locales/types.ts` — TypeScript types

```ts
import en from "./translations/en.json";
export type TranslationKeys = typeof en;
export type NestedKeyOf<T> = T extends object
  ? { [K in keyof T]: K extends string
      ? T[K] extends object ? `${K}.${NestedKeyOf<T[K]>}` : K
      : never }[keyof T]
  : never;
export type TranslationKey = NestedKeyOf<TranslationKeys>;
```

## Usage in Components

```tsx
import { useI18n } from '@/hooks/useI18n';

export default function MyScreen() {
  const { t, locale, setLocale, isRTL } = useI18n();

  return (
    <View>
      <Text>{t('common.welcome')}</Text>
      <Text>{t('audit.stepOf', { current: 1, total: 6 })}</Text>
      {isRTL && <Text>This renders only in Arabic</Text>}
      <Button onPress={() => setLocale('fr')}>Français</Button>
    </View>
  );
}
```

`useI18n()` subscribes to the event emitter. When `setLocale()` is called,
the component automatically re-renders with new translations.

## Usage Outside React (hooks, utils, constants)

```ts
import { t } from '@/locales';

// t() reads from the live i18n singleton — always returns current locale
const label = t('settings.title');
const greeting = t('home.welcomeBack');
```

Safe in event handlers, non-React hooks, constants, and module-level code.
The `t()` function always reads `i18n.locale` at call time.

### Interpolation

Use `{{variable}}` syntax for dynamic values:

```json
"reportsSummary": "{{submitted}} submitted · {{verified}} verified · {{pending}} pending"
"codeSentTo": "Code sent to {{email}}"
"stepOf": "Step {{current}} of {{total}}"
"starsCount": "{{count}}/5 stars"
```

Pass variables as the second argument to `t()`:

```ts
t('home.reportsSummary', { submitted: 12, verified: 5, pending: 3 })
```

## Adding a New Language

### Step 1: Create translation file

```bash
# Create the JSON file
touch apps/mobile/src/locales/translations/de.json
```

### Step 2: Add to i18n singleton

```ts
// src/locales/index.ts
import de from "./translations/de.json";

const SUPPORTED = ["en", "fr", "ar", "de"];
export const i18n = new I18n({ en, fr, ar, de });
```

### Step 3: Add to language picker

```tsx
// src/app/profile/language.tsx — add to options array
{
  label: t("language.german"),   // add key to language.* namespace
  value: "de",
  subtitle: "Deutsch",
},
```

### Step 4: Add translation keys to the language namespace

```json
// en.json → "language"
"german": "Deutsch"
```

## Adding Translation Keys

1. Add the key to `en.json` (source of truth)
2. Add matching key to `fr.json` and `ar.json`
3. Use in a component:

```tsx
const { t } = useI18n();
<Text>{t('feature.newKey')}</Text>
```

Or outside React:

```ts
import { t } from '@/locales';
const msg = t('feature.newKey');
```

## RTL Support

Arabic is the only RTL language. When the user switches to Arabic:

1. `I18nManager.allowRTL(true)` is called
2. `I18nManager.forceRTL(true)` is called
3. An `Alert` prompts the user to restart the app

**This is a React Native limitation** — `I18nManager` changes only take effect
after a full app restart. The restart prompt is shown automatically.

When switching *away* from Arabic:
1. `I18nManager.allowRTL(false)` is called
2. `I18nManager.forceRTL(false)` is called
3. Restart prompt is shown again

### RTL in components

```tsx
const { isRTL } = useI18n();

<View style={{ flexDirection: isRTL ? 'row-reverse' : 'row' }}>
  <Icon name="chevron-back" />
  <Text>{t('settings.title')}</Text>
</View>
```

## Anti-Patterns

### ❌ `t()` at module scope (frozen at import time)

```ts
// BAD: t() runs once at import, never updates
const title = t('settings.title');

// GOOD: call t() inside the component render
export default function MyScreen() {
  const { t } = useI18n();
  return <Text>{t('settings.title')}</Text>;
}
```

### ❌ `t()` in default props (frozen)

```ts
// BAD: default value captured once, never re-evaluated
const MyComponent = ({ label = t('common.loading') }) => ...

// GOOD: use t() inside render
const MyComponent = () => {
  const { t } = useI18n();
  return <Text>{label ?? t('common.loading')}</Text>;
};
```

### ❌ Static `t()` import in components (no re-render)

```tsx
// BAD: t from locales doesn't trigger re-render
import { t } from '@/locales';
const MyScreen = () => <Text>{t('settings.title')}</Text>;

// GOOD: t from useI18n triggers re-render on locale change
import { useI18n } from '@/hooks/useI18n';
const MyScreen = () => {
  const { t } = useI18n();
  return <Text>{t('settings.title')}</Text>;
};
```

### ❌ Using react-i18next (71 KB vs 6 KB)

The project intentionally uses `i18n-js` (~6 KB) with a custom event emitter.
`react-i18next` is ~71 KB and requires a `<Provider>` wrapper — neither
adds value here.

### ❌ Mixing AsyncStorage and SecureStore

The project uses `expo-secure-store` consistently for the `app-locale` key.
Do not use `AsyncStorage` for locale persistence — it creates split state.

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| Language switch doesn't update UI | Component uses static `t()` import | Switch to `useI18n()` hook |
| Text stays in old language after switch | `t()` called at module scope or default props | Move `t()` call inside render |
| RTL layout doesn't apply after switching to Arabic | App not restarted | Restart the app (shown in alert) |
| Translation key shows raw key string | Key missing in JSON or typo | Check all 3 JSON files for the key |
| TypeScript error on `t('invalid.key')` | Key not in `en.json` | Add the key to `en.json` first, types auto-generate |
